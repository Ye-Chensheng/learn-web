# 🦞 vibe-healing

> 你的健康陪伴搭子 —— 不是帮你短期减肥，而是帮你在高压工作与有限时间里，维持一种可持续的健康生活方式。

## 📖 产品介绍

vibe-healing 是一个 **AI 驱动的健康陪伴系统**，专为高压年轻打工人设计。

### 核心理念

- **不是** 冷冰冰的打卡工具
- **不是** 制造焦虑的减肥应用
- **是** 理解你现实处境的健康搭子
- **是** 会根据你状态调整要求的智能助手

### 核心价值

| 传统健康 App | vibe-healing |
|-------------|--------------|
| 强调计划执行 | 强调现实适配 |
| 用户单向使用工具 | 用户与 AI 双向互动 |
| 打卡导向 | 陪伴导向 |
| 偏冷静数据管理 | 偏拟人理解与鼓励 |
| 短期目标明显 | 长期生活方式导向 |

### 核心特性

| 特性 | 说明 |
|------|------|
| 🤖 AI 陪伴聊天 | 像朋友一样聊天，理解你的状态和情绪 |
| 📊 健康记录 | 轻量记录饮食、运动、睡眠、心情 |
| 🎯 动态建议 | 根据工作强度自动调整计划（保底/稳态/进阶） |
| 📈 周报反馈 | 每周总结，看到自己的进步 |
| 💬 聊天即记录 | 聊天内容自动结构化存储 |
| 🔄 数据反哺 | 小程序记录反向提供给 AI 聊天 |

## 📊 数据模型

### 核心数据表

- `users` - 用户基础信息

- `weeklyPlans` - 周计划

- 久坐型

- 晚睡型

- 情绪进食风险
- 轻断续坚持型
- 适合鼓励式沟通
- 讨厌过度提醒

## 📱 界面预览

### 页面结构

1. **首页** - 今日状态、AI 建议、连续打卡、快捷记录
2. **记录页** - 详细健康数据记录（工作强度/精力/心情/睡眠/喝水/步数）
3. **聊天页** - 与 AI 陪伴对话
4. **我的** - 个人信息、数据统计、关于

### 三种模式

| 模式 | 适用场景 | 建议策略 |
|------|---------|---------|
| 保底模式 | 加班/出差/疲惫/低能量 | 降低目标，优先维持基本健康行为 |
| 稳态模式 | 正常上班节奏 | 安排轻中度饮食与运动计划 |
| 进阶模式 | 时间充裕、状态较好 | 提供系统性训练和饮食提升方案 |

## 🎨 设计理念

### 陪伴感

- 温和、不说教的语气
- 理解用户现实处境
- 允许低谷期存在
- 奖励稳定而非极端

### 低门槛

- 快捷记录，一键完成
- 聊天式交互
- 不强制每日打卡
- 支持"保底模式"

### 价值观落地

- ✅ 允许低谷期存在（不中断惩罚）
- ✅ 奖励稳定，而不是极端
- ✅ 不制造羞耻（不用"失败"等词）
- ✅ 强调现实适配（忙时建议更轻）
- ✅ 长期成长视角（周变化、月趋势）

## 📋 产品路线图

### 阶段一：MVP 验证期 (0～3 个月)

- 跑通聊天陪伴体验
- 跑通记录结构化入库
- 跑通周报反馈闭环
- 找到用户最喜欢的使用场景

**验证指标**: 次日留存、7 日留存、每周聊天频次、周报查看率

### 阶段二：增长迭代期 (3～9 个月)

- 场景化知识库
- 轻社区
- 计划精细化
- 提醒策略优化
- 用户分层运营

### 阶段三：快速成长与商业模式验证期 (9～18 个月)

- 会员订阅
- 轻咨询服务
- 场景化课程/计划包
- 企业健康福利

### 阶段四：规模化阶段 (18 个月以后)

- 接入穿戴设备
